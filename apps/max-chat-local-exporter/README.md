# MAX Chat Local Exporter

Локальное расширение Chrome для экспорта подгруженной переписки из `https://web.max.ru/` в рабочий ZIP-архив: структурированные сообщения + картинки, привязанные к конкретным сообщениям.

## Что изменилось в версии 0.4.0

Главная задача версии 0.4.0 — сохранить не только связь текст ↔ картинка, но и reply-цепочки:

> сообщение «Петров В.А., банкрот, не подаем» → вложение `attachments/msg_0042/att_01.jpg` → reply на предыдущее сообщение/анкету, если MAX показал цитату в DOM

Внутри архива создаётся папка с такой структурой:

```text
MAX_CHAT_EXPORT_.../
  README.txt
  index.html
  messages.json
  messages.csv
  messages.txt
  attachments_manifest.csv
  attachments_manifest.csv
  attachments/
    msg_0001/
      att_01.jpg
    msg_0002/
      att_01.png
      att_02.jpg
```


## Reply-цепочки

В `messages.json` у каждого сообщения есть:

```json
{
  "bodyText": "текст самого сообщения без цитаты reply, если её удалось отделить",
  "reply": {
    "detected": true,
    "text": "цитата/превью сообщения, на которое отвечали",
    "targetMessageNumber": 41,
    "resolution": "matched_by_visible_quote_text"
  }
}
```

Если цель не найдена среди экспортированных сообщений, `targetMessageNumber` будет пустым, но сама цитата reply сохранится. Это важно: даже если старое сообщение не подгрузилось, текст цитаты всё равно останется в экспорте.

## Как понять, какая картинка к какому сообщению

### Самый удобный способ

Открой `index.html` из архива. Там каждое сообщение показано отдельным блоком:

1. номер сообщения;
2. дата/время, если MAX показал их в DOM;
3. автор/направление, если удалось определить;
4. текст сообщения;
5. вложения прямо под этим же текстом.

### Для таблиц и обработки

- `messages.csv` — одна строка = одно сообщение. В колонке `attachment_paths` перечислены вложения этого сообщения.
- `attachments_manifest.csv` — одна строка = одно вложение. Есть `message_index` и `message_text_snippet`, чтобы быстро понять, к какому тексту относится фото.
- `messages.json` — самый полный файл: у каждого сообщения есть массив `attachments`.

## Как пользоваться

1. Распакуй ZIP-архив расширения в отдельную папку.
2. Открой Chrome → `chrome://extensions`.
3. Включи «Режим разработчика».
4. Нажми «Загрузить распакованное расширение».
5. Выбери папку `max-chat-local-exporter`.
6. Открой `https://web.max.ru/`, зайди в нужный чат.
7. Нажми иконку расширения → «Показать панель».
8. Нажми «Автопрокрутка вверх» и дождись остановки.
9. Нажми «ZIP: сообщения + картинки».
10. Распакуй полученный ZIP и открой `index.html`.

## Важные ограничения

Это не официальный экспорт MAX. Расширение читает DOM веб-страницы и сохраняет то, что веб-версия MAX уже подгрузила.

Из этого следуют ограничения:

- если старые сообщения не были подгружены прокруткой, их не будет в архиве;
- если MAX показывает только миниатюру, а полный файл не доступен через URL в DOM, может сохраниться миниатюра;
- если картинка лежит на CDN, который Chrome не даст скачать расширению, в архиве будет `_FAILED.txt` рядом с сообщением и причина ошибки;
- если MAX виртуализирует список и удаляет старые сообщения из DOM, нужно запускать автопрокрутку и не переключать чат до завершения.

## Безопасность

Расширение не отправляет переписку на серверы разработчика и не содержит аналитики. Оно делает только две вещи:

1. читает DOM открытого `web.max.ru`;
2. скачивает изображения, ссылки на которые уже найдены в открытом чате, чтобы положить их в локальный ZIP.

В `manifest.json` есть широкое `host_permissions: https://*/*`. Это нужно не для чтения всех сайтов, а чтобы background script мог скачать картинки, если MAX отдаёт их с CDN вне `web.max.ru`. Content script всё равно запускается только на `https://web.max.ru/*`.

## Быстрая проверка кода

Открой файлы расширения и проверь:

- нет `sendBeacon`;
- нет сторонних библиотек;
- нет адресов внешних серверов для отправки данных;
- `background.js` используется только для скачивания URL вложений и возвращает данные обратно в расширение.

## Что делать, если фото не сохранились

1. Открой чат и кликни по фото, чтобы MAX подгрузил его крупнее.
2. Закрой просмотрщик фото, не перезагружая страницу.
3. Нажми «Сканировать экран».
4. Снова нажми «ZIP: сообщения + картинки».

Если рядом с сообщением появился файл `att_01_FAILED.txt`, открой его: там будет URL/причина ошибки и текст сообщения, к которому относилось вложение.


## v0.4.1

Добавлено извлечение ссылок на сообщения MAX, если web.max.ru показывает permalink или хранит его в DOM. В `messages.json` появляются поля `messageUrl`, `messageUrlSource`, `messageLinkCandidates`, `maxChatUrl`, `localExportAnchor`. В `messages.csv` добавлены колонки `message_url`, `message_url_source`, `max_chat_url`, `local_export_anchor`. Если MAX не отдаёт прямую ссылку на конкретное сообщение, поле `messageUrl` останется пустым, а `maxChatUrl` сохранит ссылку на текущий чат/страницу.

## Имена архивов

ZIP теперь называется автоматически по схеме:

```text
MAX_CHAT_EXPORT_1698msg_1149att_22-06-26_23-49.zip
```

Где:

- `1698msg` — количество сообщений, попавших в экспорт;
- `1149att` — количество вложений/картинок-кандидатов на анкеты;
- `22-06-26_23-49` — локальная дата и время создания архива в формате `дд-мм-гг_чч-мм`.

Точное количество именно распознанных анкет появляется после OCR в `forms_structured.json` / `forms_flat.csv`. На этапе Chrome-экспорта расширение честно считает вложения, потому что ещё не знает, анкета это или другая картинка.
